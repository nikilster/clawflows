# Remember Conversations

People remember when you remember. After an important meeting, call, or chat, this workflow helps you capture the things that matter — promises you made, things you learned about someone, and action items — so you can follow through and build stronger relationships.

## Steps

1. Ask the user who they spoke with, when, and what the general topic was. Check your **calendar skill** for recent meetings if the user isn't sure.
2. Ask for the key highlights: what was discussed, any commitments made (by either side), and any personal details worth remembering.
3. Organize the notes into a clean summary with sections for takeaways, action items, and personal details.
4. Save the summary as a note using your **notes skill** or file skill, tagged with the person's name and date.
5. If there are action items with deadlines, offer to add them to the calendar or reminders.

## Output Format

```
🧠 Conversation Log — March 18, 2026

With: Jordan (college friend)
Context: Caught up over coffee

Key Takeaways:
- Jordan is switching careers from finance to UX design
- Starting a bootcamp in April
- Mentioned feeling nervous about the transition

Promises Made:
- I said I'd intro them to my friend Sam who works in UX
- Jordan offered to help me with my tax questions

Personal Details:
- Partner's name: Alex
- Dog's name: Biscuit (golden retriever)
- Moving to a new apartment in May
```

## Notes

- Keep notes factual and respectful — these are personal details someone trusted you with.
- Don't store anything sensitive (health issues, financial details) unless the user explicitly wants to.
- Over time, you can reference past logs before future meetings with the same person to help the user prepare.
- This workflow is meant to be run right after a conversation while details are fresh.
