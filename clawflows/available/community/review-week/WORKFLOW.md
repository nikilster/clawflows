# Weekly Review

End-of-week review of your calendar, meetings, tasks, and goals.

## 1. Review Calendar Events

Using your **calendar skill**, pull all events from Monday through today:
- Total meetings and hours spent in meetings
- 1-on-1s vs group meetings vs external meetings
- Meetings that led to decisions or action items
- Events cancelled or skipped

Summarize the week's meeting load in 2-3 sentences.

## 2. Review Completed Tasks

Using your **task manager skill** (if available):
- Tasks completed this week
- Tasks still in progress
- Overdue or blocked items and why

If no task system is available, note it for the user to fill in.

## 3. Check Goals Progress

Ask the user about current goals (or check notes from previous runs):
- Progress on weekly or monthly goals
- Milestones reached

## 4. Compile the Review

```markdown
# Week of YYYY-MM-DD

## Highlights
- [Top 3 accomplishments or moments]

## Meetings
- X meetings (Y hours)
- [Key outcomes or decisions]

## Tasks
- X completed, Y in progress, Z blocked

## Goals Progress
- [Status on current goals]

## Next Week
- [Top priorities and key deadlines]

## Reflection
[What went well, what to improve]
```

## 5. Save and Present

Save to `~/Documents/reviews/week-of-YYYY-MM-DD.md` or the user's preferred location.

## Notes

- Focus on outcomes and decisions, not activity counts.
- Keep the review to one page. Brevity is a feature.
- If it was a light week (PTO, sick), say so honestly.
- Use past reviews to spot trends over time.
